{\ttfamily hybstr} provides a {\bfseries{hybrid string type}} ({\ttfamily \doxylink{classhybstr_1_1string__impl}{hybstr\+::string\+\_\+impl}}) that seamlessly bridges {\bfseries{compile-\/time}} ({\ttfamily constexpr}) and {\bfseries{runtime}} string handling.

Unlike {\ttfamily std\+::string} (purely runtime) or {\ttfamily std\+::string\+\_\+view} (non-\/owning), {\ttfamily \doxylink{classhybstr_1_1string__impl}{hybstr\+::string\+\_\+impl}} owns its data and can operate in both environments.

In both compile-\/time and runtime enviroments, all data lives in a fixed buffer known to the compiler.

{\ttfamily hybstr} lets you treat strings as {\bfseries{first-\/class compile-\/time objects}} â€” useful for metaprogramming, code generation, embedded DSLs, or constexpr evaluation â€” while still functioning as a normal string type at runtime.\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md__r_e_a_d_m_e_autotoc_md2}

\begin{DoxyItemize}
\item {\bfseries{Hybrid compile-\/time / runtime}} Works in both {\ttfamily constexpr} and regular runtime expressions.
\item {\bfseries{Concatenation and modification}} Supports operations like {\ttfamily +}, {\ttfamily append, push\+\_\+back}, and {\ttfamily set\texorpdfstring{$<$}{<}N\texorpdfstring{$>$}{>}()} in both compile time and runtime.
\item {\bfseries{NTTP}} Can be used as a Non-\/\+Type Template Parameter (NTTP) in C++20 and above.
